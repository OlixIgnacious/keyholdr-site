How can we
help you?

Keyholdr is built and maintained by a solo developer. If you're running into an issue or have a question, reach out directly — responses usually arrive within a day or two.

Common questions

  • The hotkey isn't working — make sure Keyholdr is running (look for the key icon in your menu bar). On macOS Sequoia, you may need to re-grant Accessibility permissions in System Settings → Privacy & Security → Accessibility.
  • Touch ID keeps asking for permission — on first use, macOS shows a Keychain access prompt. Choose "Always Allow" and it won't ask again.
  • CLI not found after install — open a new terminal window after installing. If it still isn't found, add export PATH="$HOME/.local/bin:$PATH" to your shell profile manually.
  • CLI not found (Mac App Store version) — the App Store build is sandboxed and can't link the CLI onto your PATH by itself. Open the app, go to the ⋯ menu → Terminal Setup…, and copy the command shown there (or see the install docs).
  • Restoring keys after reinstall — your secrets are stored in macOS Keychain, so they survive reinstalls as long as you stay on the same Mac and account. Use File → Export Vault before switching Macs.
